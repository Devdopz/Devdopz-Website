# Security Policy

## Supported Versions

Security fixes are expected to be applied to the latest code on `main` and the
latest production deployment of the site.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Latest production deployment | Yes |
| Older snapshots and forks | No guarantee |

## Reporting a Vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Report security concerns privately using:

- Email: `teamdevdopz@gmail.com`

When possible, include:

- a clear description of the issue
- steps to reproduce
- affected URLs, routes, or files
- potential impact
- screenshots or proof of concept if safe to share

## What to Expect

We aim to:

- acknowledge good-faith reports in a reasonable time
- investigate the issue and validate impact
- work on a fix or mitigation
- coordinate disclosure when appropriate

## Responsible Disclosure

Please avoid:

- public disclosure before a fix is available
- accessing data that does not belong to you
- destructive testing or denial-of-service activity
- social engineering, spam, or privacy-invasive behavior

We appreciate responsible disclosure that helps keep the project and its users
safe.
